# Переменные окружения

Переменные окружения — это значения, которые операционная система передаёт программе во время запуска.

Проще говоря, это способ сказать программе:

- где искать файлы;
- на каком порту запускаться;
- какой режим использовать: разработка или продакшен;
- какой ключ доступа или пароль применять.

## Простая аналогия

Представь, что программа — это человек, который пришёл на работу.
Переменные окружения — это записка на столе с важной информацией:

- твой кабинет — `HOME`;
- где лежат инструменты — `PATH`;
- в каком режиме сегодня работать — `NODE_ENV`;
- какой у тебя пропуск — `API_KEY`.

Сама программа обычно не хранит всё это жёстко внутри кода, а читает из окружения.

## Зачем они нужны

### 1. Чтобы не менять код под каждую среду

Например, локально приложение работает с одной базой данных, а на сервере — с другой.
Код остаётся тем же, меняются только значения переменных окружения.

### 2. Чтобы хранить настройки отдельно от программы

Это делает проект удобнее:

- код отвечает за логику;
- переменные окружения отвечают за конфигурацию.

### 3. Чтобы не писать секреты прямо в коде

Пароли, токены и ключи лучше не хранить в репозитории.
Часто их передают через переменные окружения.

## Примеры

- `PATH` — показывает, где система ищет исполняемые файлы;
- `HOME` — домашняя папка пользователя;
- `PORT=3000` — порт, на котором стартует сервер;
- `DEBUG=true` — включает дополнительный вывод для отладки;
- `API_KEY=...` — ключ для доступа к внешнему сервису.

## Пример с PATH и Go

Когда ты устанавливаешь некоторые Go-инструменты, их исполняемые файлы попадают в каталог:

```bash
$(go env GOPATH)/bin
```

Чтобы система могла запускать эти команды из любого места в терминале, этот путь добавляют в `PATH`.

Например: файл `$HOME/.zshrc` может содержать строку:

```bash
export PATH="$PATH:$(go env GOPATH)/bin"
```

Что здесь происходит:

- `$PATH` — текущее значение переменной `PATH`;
- `:` — разделитель путей в macOS и Linux;
- `$(go env GOPATH)/bin` — папка, где Go хранит установленные утилиты.

После этого система будет искать команды не только в старых папках из `PATH`, но и в папке Go-инструментов.

Например, если ты установил утилиту вроде `golangci-lint` или `air`, терминал сможет найти её по имени команды.

Если нужно сохранить это поведение между перезапусками терминала, такую строку обычно добавляют в файл конфигурации оболочки, например в `.zshrc`.

## Как программа их использует

Обычно программа запрашивает значение по имени переменной.

Например:

- если есть `PORT`, сервер запускается на этом порту;
- если `DEBUG=true`, приложение пишет больше логов;
- если нет `API_KEY`, запросы к внешнему API могут не работать.

## Где их задают

Переменные окружения можно задавать по-разному:

- в терминале перед запуском программы;
- в настройках системы;
- в файле `.env`, если проект использует такой формат;
- в настройках хостинга, Docker, CI/CD или облачной платформы.

## Что важно помнить новичку

- Переменная окружения — это обычно строка.
- Даже если там написано `true`, `false` или `3000`, программе часто нужно самой преобразовать это значение в нужный тип.
- Если переменная не задана, программа может завершиться с ошибкой или взять значение по умолчанию.

## Хорошая практика

- не храни секреты прямо в коде;
- не коммить `.env`, если в нём есть токены и пароли;
- документируй, какие переменные нужны проекту;
- задавай безопасные значения по умолчанию там, где это возможно.

## Коротко

Переменные окружения — это внешний способ передать программе настройки без изменения её кода.
Это один из базовых инструментов конфигурации, который постоянно используется в разработке, администрировании и computer science-практике вокруг запуска программ.
